
### 4.1.3 Firewalls

Un firewall esta diseñado para controlar o filtrar que comunicaciones se permiten dentro y cuales se permiten fuera de un dispositivo o red. Se puede proteger un solo equipo con el firewall basado en host o puede ser uno que proteja toda la red que es un firewall basado en red.

Firewalls mas comunes:

- Firewall de capa de red: filtra comunicaciones según las direcciones ip de origen y destino.   
- Firewall de capa de transporte: filtra según los puertos de datos de origen y destino asi coo los estados de conexión.   
- De capa de aplicación: filtra en función de una aplicación, programa o servicio  
- de capa sensible al contexto: filtra comunicaciones según el usuario, dispositivo, función, tipo de aplicación y perfil de amenaza.   
- Servidor proxy: filtra las solicitudes de contenido web, como url, nombres de dominio y tipos de medios.   
- servidor proxy inverso: colocados frente a servidores web, los servidores proxy inversos, protegen ocultan, descargan y distribuyen el acceso a los servidores web   
- firewalls de nat: oculta o enmascara direcciones privadas de los hosts de la red.   
- Basado en el host: filtra puertos y llamadas de servicio del sistema en un Oslo sistema operativo de computadora.

### 4.2.3 Pruebas de penetracion

Son el acto de evaluar un sistema informatico, una red, o una organización en busca de vulnerabilidades de seguridad. Esta busca violar sistemas, personas, procesos y códigos para descibrir vulnerabilidades.

Prueba de lapiz en 5 pasos

1. Planificación: recopila la mayor cantidad de información posible sobre un sistema o red de destino, llevar reconocimineot activo o pasivo.   
2. Escaneo: lleva a cabo un reconocimiento activo para sondear un sistema o red objetivo e identificar posibles debilidades que, si se explotan, podrían dar acceso a un atacante.   
3. Obtener acceso: obtener acceso a un sistema de destino y rastreará el tráfico de la red, utilizando varios métodos para explotar el sistema.   
4. Mantener el Acceso: mantendrá el acceso al objetivo para averiguar qué datos y sistemas son vulnerables a la explotación.   
5. Análisis y reportes: proporcionará comentarios a través de un informe que recomienda actualizaciones de los productos, las políticas y la capacitación para mejorar la seguridad de la organización.
